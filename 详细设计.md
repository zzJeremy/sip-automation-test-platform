# SIP自动化测试平台详细设计

## 1. 模块详细设计

### 1.1 统一接口层（SIPClientBase）

#### 1.1.1 类设计
```python
class SIPClientBase(ABC):
    def __init__(self, config: dict):
        """初始化SIP客户端"""
        
    @abstractmethod
    def register(self, user: str, password: str, domain: str) -> bool:
        """用户注册"""
        
    @abstractmethod
    def make_call(self, from_user: str, to_user: str, domain: str) -> bool:
        """发起呼叫"""
        
    @abstractmethod
    def send_message(self, from_user: str, to_user: str, domain: str, message: str) -> bool:
        """发送消息"""
        
    @abstractmethod
    def unregister(self) -> bool:
        """注销"""
        
    @abstractmethod
    def close(self) -> None:
        """关闭客户端"""
```

#### 1.1.2 接口规范
- 所有方法需实现超时控制
- 统一错误码定义
- 支持异步操作

### 1.2 协议实现层

#### 1.2.1 SIPpDriverClient
- **职责**：作为SIPp的驱动适配器，负责生成场景文件、启动SIPp进程、监控状态、收集报告
- **实现细节**：
  - 通过命令行接口与SIPp交互
  - 生成SIPp XML场景文件
  - 监控SIPp进程状态
  - 解析SIPp输出日志和报告
- **依赖**：SIPp工具

#### 1.2.2 PJSIPSIPClient
- **职责**：基于PJSIP库的专业SIP客户端，支持复杂SIP场景和媒体处理
- **实现细节**：
  - 使用PJSIP库实现SIP协议栈
  - 支持完整媒体流处理
  - 实现安全协议（TLS/SRTP）
- **依赖**：PJSIP库

#### 1.2.3 SocketFuzzClient
- **职责**：基于socket的模糊测试客户端，专注于畸形报文和安全性测试
- **实现细节**：
  - 直接使用socket发送自定义SIP消息
  - 支持生成任意格式的SIP报文
  - 实现协议边界测试
- **依赖**：Python socket模块

#### 1.2.4 SocketSIPClientAdapter
- **职责**：适配器模式，将原有socket实现适配到统一接口
- **实现细节**：
  - 封装原有socket实现
  - 适配到SIPClientBase接口
- **依赖**：Python socket模块

### 1.3 智能调度层（TestOrchestrator）

#### 1.3.1 类设计
```python
class TestOrchestrator:
    def __init__(self, clients: List[SIPClientBase]):
        """初始化调度器"""
        
    def select_client(self, test_type: str) -> SIPClientBase:
        """根据测试类型选择客户端"""
        
    def execute_test(self, test_case: dict) -> dict:
        """执行测试用例"""
        
    def handle_failure(self, client: SIPClientBase, fallback: bool = True) -> bool:
        """处理客户端故障"""
```

#### 1.3.2 调度策略
- 性能/压力/负载测试 → SIPpDriverClient
- 安全/负向测试 → SocketFuzzClient
- 功能/合规性测试 → PJSIPSIPClient
- 基础功能测试 → SocketSIPClientAdapter
