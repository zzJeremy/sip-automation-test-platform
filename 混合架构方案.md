# SIP自动化测试平台混合架构方案（概要设计）

## 1. 背景与目标

### 1.1 问题背景
原生socket编程在SIP自动化测试中存在局限性，但其对协议细节的完全掌控能力仍有价值。需要设计一个混合架构，既能解决socket编程的局限性，又能保留其优势。

### 1.2 设计目标
- 解决纯socket编程在SIP测试中的复杂度爆炸问题
- 保留socket编程对协议细节的完全掌控能力
- 实现多种SIP客户端实现的统一接口
- 提供智能调度机制，根据测试类型选择最适合的客户端
- 确保架构的可扩展性和可维护性

## 2. 架构设计原则

### 2.1 统一接口原则
- 定义统一的SIP客户端接口，屏蔽底层实现差异
- 确保不同客户端实现间的无缝切换

### 2.2 职责分离原则
- 接口层：定义统一的SIP操作接口
- 实现层：提供多种SIP客户端实现
- 调度层：根据测试需求智能选择实现

### 2.3 渐进式演进原则
- 保留现有socket实现，逐步引入新功能
- 确保向后兼容性

## 3. 混合架构设计

### 3.1 三层架构模型

```
+-----------------------------------------+
|              统一接口层                  |
|        SIPClientBase (统一接口)         |
+-----------------------------------------+
|              智能调度层                  |
|          TestOrchestrator               |
|  根据测试类型自动选择最合适的客户端实现  |
+-----------------------------------------+
|              协议实现层                  |
|  +----------------+ +----------------+  |
|  |  SIPpDriver    | |   PJSIPSIP     |  |
|  |   Client       | |   Client       |  |
|  |(标准场景/性能)  | |(复杂SIP/安全) |  |
|  +----------------+ +----------------+  |
|                                         |
|  +----------------+ +----------------+  |
|  | SocketFuzz     | | MultiInstance  |  |
|  |   Client       | |  Coordinator   |  |
|  |(Fuzz/安全测试)  | |(状态协调)     |  |
|  +----------------+ +----------------+  |
+-----------------------------------------+
```

### 3.2 统一接口层设计
- 定义标准SIP操作接口（注册、呼叫、消息、注销等）
- 提供统一的错误处理机制
- 实现统一的配置管理

### 3.3 协议实现层设计
- **SIPpDriverClient**: 作为SIPp的驱动适配器，负责生成场景文件、启动SIPp进程、监控状态、收集报告
- **PJSIPSIPClient**: 基于PJSIP库的专业SIP客户端，支持复杂SIP场景和媒体处理
- **SocketFuzzClient**: 基于socket的模糊测试客户端，专注于畸形报文和安全性测试
- **SocketSIPClientAdapter**: 适配器模式，将原有socket实现适配到统一接口

### 3.4 智能调度层设计
- 根据测试用例标签自动选择最合适的客户端实现
- 支持优先级排序和故障回退
- 提供测试结果统计和分析

## 4. 调度策略

### 4.1 测试类型映射
- 性能/压力/负载测试 → SIPpDriverClient
- 安全/负向测试 → SocketFuzzClient
- 功能/合规性测试 → PJSIPSIPClient
- 基础功能测试 → SocketSIPClientAdapter

### 4.2 优先级策略
- 按测试类型确定客户端优先级
- 支持故障时自动回退到备选客户端
- 提供负载均衡机制

## 5. 扩展性设计

### 5.1 新客户端实现扩展
- 通过继承SIPClientBase实现新客户端
- 自动集成到调度系统中

### 5.2 新测试类型扩展
- 通过扩展TestType枚举添加新测试类型
- 更新调度策略以支持新类型

## 6. 部署与集成

### 6.1 依赖管理
- 可选依赖：PJSIP库、SIPp工具
- 核心依赖：Python标准库、socket模块

### 6.2 配置管理
- 统一配置接口
- 支持环境特定配置
