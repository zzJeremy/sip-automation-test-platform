# SIP自动化测试平台概要设计

## 1. 项目概述

### 1.1 项目背景
SIP自动化测试平台（AutoTestForUG）是一个基于pytest框架的SIP协议自动化测试系统，采用"开源测试框架 + 轻量领域层（DSL/adapter）"的混合架构。该系统充分利用pytest的生态系统，同时在pytest之上实现SIP特定的领域层，提供简洁的API来定义SIP测试场景，旨在解决传统SIP自动化测试方案的复杂性和维护性问题。

### 1.2 项目目标
- 利用pytest成熟的测试执行和管理能力
- 提供SIP协议特定的领域特定语言（DSL）
- 支持多种SIP测试场景和配置方式
- 确保系统的可扩展性和可维护性
- 提供灵活的测试定义和执行策略

## 2. 系统架构设计

### 2.1 总体架构
系统采用混合架构设计，结合pytest框架与SIP特定领域层：

```
┌─────────────────────────────────────────┐
│        领域特定语言层（DSL）             │
│      SIPCallFlow, SIPMessageValidator   │
└─────────────────────────────────────────┘
                │
┌─────────────────────────────────────────┐
│          测试定义层                      │
│        YAML/JSON 测试用例               │
└─────────────────────────────────────────┘
                │
┌─────────────────────────────────────────┐
│          资源管理层                      │
│      pytest fixtures (端口池、客户端等)    │
└─────────────────────────────────────────┘
                │
┌─────────────────────────────────────────┐
│          执行引擎层                      │
│           pytest 框架                   │
└─────────────────────────────────────────┘
```

### 2.2 架构设计原则
- **利用成熟框架原则**：充分利用pytest生态系统的优势
- **领域特定语言原则**：提供简洁的API定义SIP测试场景
- **资源管理原则**：通过pytest fixtures实现自动化资源管理
- **可扩展性原则**：支持新增测试类型和扩展功能
- **配置分离原则**：将测试逻辑与配置分离，支持多种定义方式

## 3. 系统模块划分

### 3.1 pytest测试套件（pytest_sip_tests）
- **fixtures.py**: 提供核心pytest fixtures，管理端口池、客户端实例、测试环境等共享资源
- **sip_dsl.py**: 定义SIP特定的领域特定语言，提供简洁的API定义SIP测试场景
- **test_case_parser.py**: 解析YAML/JSON格式的测试用例定义
- **report_generator.py**: 生成多格式测试报告（JSON, HTML, Text）
- **test_executor.py**: 测试执行器，集成pytest运行并提供命令行接口
- **conftest.py**: pytest配置文件，定义标记和全局设置

### 3.2 领域特定语言层（DSL）
- **SIPCallFlow**: 定义SIP呼叫流程的DSL，支持流式调用风格
- **SIPMessageValidator**: SIP消息验证器，提供灵活的消息匹配和验证功能
- **ScenarioBuilder**: 测试场景构建器，支持复杂SIP交互场景定义
- **AssertionLibrary**: SIP特定的断言库，提供协议级别的验证功能

### 3.3 资源管理层
- **PortPool**: 端口池管理，确保测试间端口资源不冲突
- **SIPClientFactory**: SIP客户端工厂，创建和管理SIP客户端实例
- **TestEnvironment**: 测试环境管理，支持不同测试模式（真实环境 vs 模拟环境）
- **PCAPCollector**: 网络包收集器，捕获SIP交互过程中的网络流量

## 4. 系统功能设计

### 4.1 核心功能
- SIP协议消息的发送和接收
- 会话建立、维护和释放
- 测试用例执行和结果收集
- 错误处理和异常恢复
- 测试报告生成

### 4.2 扩展功能
- 多格式测试报告（JSON, HTML, Text）
- 并行测试执行支持
- YAML/JSON格式测试用例定义
- pytest标记和参数化支持
- 灵活的资源管理

## 5. 接口设计

### 5.1 外部接口
- 命令行接口：通过test_executor.py提供统一的测试执行入口
- 配置文件接口：支持YAML/JSON格式的测试配置
- 报告接口：支持多格式测试结果导出

### 5.2 内部接口
- pytest fixtures接口：用于资源管理和共享
- DSL接口：SIP特定的API定义
- 测试执行接口：与pytest框架的集成接口

## 6. 数据设计

### 6.1 数据结构
- SIP消息结构：定义SIP消息的通用格式
- 测试结果结构：定义测试结果的数据格式
- 配置参数结构：定义系统配置参数
- 测试用例结构：定义YAML/JSON格式的测试用例

### 6.2 数据流
- 测试用例数据流：从YAML/JSON定义到pytest参数化测试的转换流程
- 配置数据流：配置参数在系统各层间的传递
- 报告数据流：从测试执行到报告生成的完整流程

## 7. 部署设计

### 7.1 系统部署
- 支持单机部署
- 依赖管理：Python包依赖通过requirements.txt管理
- 可选依赖：SIPp工具、SIP服务器等外部依赖

### 7.2 运行环境
- 操作系统：Windows/Linux/macOS
- 语言环境：Python 3.7+
- 网络环境：支持IPv4/IPv6
- 需要可访问的SIP服务器或SIP服务

## 8. 扩展性设计

### 8.1 DSL扩展
- 通过扩展SIP DSL类实现新的SIP操作
- 支持自定义验证器和断言
- 保持与pytest生态系统的兼容性

### 8.2 新功能扩展
- 支持新的SIP协议扩展
- 支持新的测试类型和场景
- 集成更多pytest插件（如pytest-xdist并行、pytest-rerunfailures重试、allure-pytest报告）
- 核心依赖：pytest框架、Python标准库
