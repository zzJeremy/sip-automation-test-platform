# SIP自动化测试平台前端界面设计文档

## 1. 概述

### 1.1 项目背景
SIP自动化测试平台前端界面是为SIP测试系统提供的现代化Web界面，旨在提供直观、易用的测试管理体验。该界面采用Vue 3 + Element Plus技术栈构建，支持测试套件管理、执行监控、报告查看等功能。

### 1.2 设计目标
- 提供直观的用户界面，降低使用门槛
- 支持实时监控测试执行状态
- 提供可视化测试套件管理功能
- 支持拖拽式测试场景编排
- 提供多格式报告导出功能
- 确保良好的用户体验和响应速度

### 1.3 技术栈
- Vue 3: 前端框架
- Element Plus: UI组件库
- Pinia: 状态管理
- Vue Router: 路由管理
- Axios: HTTP客户端
- WebSocket: 实时通信
- Vuedraggable: 拖拽功能
- ECharts: 图表展示

### 4.3 API通信设计
```javascript
// utils/api.js
export const suiteApi = {
  // 获取所有测试套件
  getAllSuites: (params = {}) => apiClient.get('/suites', { params }),
  
  // 创建测试套件
  createSuite: (suiteData) => apiClient.post('/suites', suiteData),
  
  // 更新测试套件
  updateSuite: (id, suiteData) => apiClient.put(`/suites/${id}`, suiteData),
  
  // 删除测试套件
  deleteSuite: (id) => apiClient.delete(`/suites/${id}`)
}

export const executionApi = {
  // 开始执行测试
  startExecution: (executionData) => apiClient.post('/executions', executionData),
  
  // 停止执行测试
  stopExecution: (id) => apiClient.delete(`/executions/${id}`),
  
  // 获取执行状态
  getExecutionStatus: (id) => apiClient.get(`/executions/${id}/status`)
}
```

### 4.4 WebSocket实时通信设计
```javascript
// utils/websocket.js
class WebSocketService {
  constructor() {
    this.ws = null
    this.store = useMainStore()
    this.isReconnecting = false
    this.reconnectInterval = 5000
  }

  // 连接WebSocket
  connect(url) {
    if (this.ws && this.ws.readyState === WebSocket.OPEN) {
      return
    }

    this.ws = new WebSocket(url)
    
    this.ws.onmessage = (event) => {
      const data = JSON.parse(event.data)
      this.handleMessage(data)
    }
  }

  // 处理接收到的消息
  handleMessage(data) {
    switch (data.type) {
      case 'log':
        // 添加日志到store
        this.store.addLog({
          time: data.timestamp || new Date().toLocaleTimeString(),
          level: data.level || 'INFO',
          message: data.message || '',
          sipMessage: data.sipMessage || null
        })
        break
      case 'executionStatus':
        // 更新执行状态
        this.updateExecutionStatus(data.status)
        break
      case 'progress':
        // 更新执行进度
        this.updateExecutionProgress(data.progress)
        break
    }
  }
}
```

## 5. 用户交互设计

### 5.1 操作流程
1. **测试套件管理流程**
   - 查看套件列表 → 选择或创建套件 → 配置套件参数 → 运行测试

2. **测试执行监控流程**
   - 选择测试套件 → 配置执行参数 → 开始执行 → 监控执行状态 → 查看结果

3. **场景编辑流程**
