# 软交换系统自动化测试平台实现文档

## 1. 项目概述

### 1.1 项目背景
软交换系统自动化测试平台（AutoTestForUG）是一个基于pytest框架的SIP协议自动化测试系统，采用"开源测试框架 + 轻量领域层（DSL/adapter）"的混合架构。该系统充分利用pytest的生态系统，同时在pytest之上实现SIP特定的领域层，提供简洁的API来定义SIP测试场景，旨在解决传统SIP自动化测试方案的复杂性和维护性问题。

### 1.2 项目目标
- 利用pytest成熟的测试执行和管理能力
- 提供SIP协议特定的领域特定语言（DSL）
- 支持多种SIP测试场景和配置方式
- 确保系统的可扩展性和可维护性
- 提供灵活的测试定义和执行策略

## 2. 系统架构设计

### 2.1 总体架构
系统采用混合架构设计，结合pytest框架与SIP特定领域层：

```
┌─────────────────────────────────────────┐
│              业务层                      │
│    BusinessTestSuite, TestScenarioManager│
└─────────────────────────────────────────┘
                │
┌─────────────────────────────────────────┐
│        领域特定语言层（DSL）             │
│      SIPCallFlow, SIPMessageValidator   │
└─────────────────────────────────────────┘
                │
┌─────────────────────────────────────────┐
│          测试定义层                      │
│        YAML/JSON 测试用例               │
└─────────────────────────────────────────┘
                │
┌─────────────────────────────────────────┐
│          资源管理层                      │
│      pytest fixtures (端口池、客户端等)    │
└─────────────────────────────────────────┘
                │
┌─────────────────────────────────────────┐
│          执行引擎层                      │
│           pytest 框架                   │
└─────────────────────────────────────────┘
```

### 2.2 架构设计原则
- **利用成熟框架原则**：充分利用pytest生态系统的优势
- **领域特定语言原则**：提供简洁的API定义SIP测试场景
- **资源管理原则**：通过pytest fixtures实现自动化资源管理
- **可扩展性原则**：支持新增测试类型和扩展功能
- **配置分离原则**：将测试逻辑与配置分离，支持多种定义方式

## 3. 系统模块划分

### 3.1 pytest测试套件（pytest_sip_tests）
- **fixtures.py**: 提供核心pytest fixtures，管理端口池、客户端实例、测试环境等共享资源
- **sip_dsl.py**: 定义SIP特定的领域特定语言，提供简洁的API定义SIP测试场景
- **test_case_parser.py**: 解析YAML/JSON格式的测试用例定义
- **report_generator.py**: 生成多格式测试报告（JSON, HTML, Text）
- **test_executor.py**: 测试执行器，集成pytest运行并提供命令行接口
- **conftest.py**: pytest配置文件，定义标记和全局设置

### 3.2 业务层（Business Layer）
- **BusinessTestSuite**：业务测试套件，组织多个相关测试
- **TestScenarioManager**：测试场景管理器，编排复杂测试流程
- **TestCaseOrchestrator**：测试用例协调器，控制测试执行顺序

### 3.3 客户端层（Client Layer）
- **SIPClientBase**：SIP客户端基类，定义通用接口
- **HybridClient**：混合客户端，支持多种SIP实现
- **ClientManager**：客户端管理器，统一管理客户端生命周期
- **DialogStateManager**：对话状态管理器，跟踪SIP对话状态

### 3.4 Robot Framework适配器（rf_adapter）
- **RobotFrameworkAdapter**：将Robot Framework测试转换为pytest执行
- **SIPRobotKeywords**：SIP相关的Robot Framework关键字
- 提供关键字驱动测试功能，让非开发人员可以使用

## 4. 核心功能实现

### 4.1 SIP DSL实现
```python
class SIPCallFlow:
    def __init__(self, caller_uri: str, callee_uri: str):
        """初始化SIP呼叫流程"""
        self.caller_uri = caller_uri
        self.callee_uri = callee_uri
        self.steps = []
        self.duration = 0
        
    def invite(self):
        """添加INVITE步骤"""
        self.steps.append({"action": "invite", "timestamp": len(self.steps)})
        return self
        
    def wait_for_200_ok(self):
        """添加等待200 OK步骤"""
        self.steps.append({"action": "wait_200_ok", "timestamp": len(self.steps)})
        return self
        
    def ack(self):
        """添加ACK步骤"""
        self.steps.append({"action": "ack", "timestamp": len(self.steps)})
        return self
        
    def wait(self, seconds: int):
        """添加等待步骤"""
        self.duration += seconds
        self.steps.append({"action": "wait", "seconds": seconds, "timestamp": len(self.steps)})
        return self
        
    def bye(self):
        """添加BYE步骤"""
        self.steps.append({"action": "bye", "timestamp": len(self.steps)})
        return self
```

## 5. 资源管理机制

### 5.1 端口池管理
```python
class PortPool:
    def __init__(self, start_port: int, end_port: int):
        """初始化端口池"""
        self.start_port = start_port
        self.end_port = end_port
        self.available_ports = set(range(start_port, end_port + 1))
        self.allocated_ports = set()
        self.lock = threading.Lock()
        
    def acquire(self) -> int:
        """获取可用端口"""
        with self.lock:
            if not self.available_ports:
                raise Exception("No available ports")
            port = self.available_ports.pop()
            self.allocated_ports.add(port)
            return port
            
    def release(self, port: int) -> bool:
        """释放端口"""
        with self.lock:
            if port in self.allocated_ports:
                self.allocated_ports.remove(port)
                self.available_ports.add(port)
                return True
            return False
```

```yaml
test_cases:
  - name: "基础呼叫建立测试"
    description: "测试基本的SIP呼叫建立流程"
    markers: ["sip_basic", "regression"]  # pytest标记
    config:
      server_host: "127.0.0.1"

      - action: "send_invite"

      - action: "wait_for_200ok"

      - action: "send_ack"
        description: "发送ACK确认"
      - action: "send_bye"
```

### 6.2 pytest格式测试用例
```python
@pytest.mark.sip_basic
@pytest.mark.parametrize("test_config", [
    {
        "caller_uri": "sip:alice@127.0.0.1:5060",
        "callee_uri": "sip:bob@127.0.0.1:5060",
        "call_duration": 5
    },
    {
        "caller_uri": "sip:charlie@127.0.0.1:5060", 
        "callee_uri": "sip:diana@127.0.0.1:5060",
        "call_duration": 3
    }
])
def test_basic_sip_call(sip_client_factory, test_config: Dict[str, Any]):
    """测试基础SIP呼叫功能"""
    caller_client = sip_client_factory(
        client_id="caller",
        local_port=test_config.get("local_port", 5080)
    )
    
    # 执行呼叫
    result = caller_client.make_basic_call(
        caller_uri=test_config["caller_uri"],
        callee_uri=test_config["callee_uri"], 
        call_duration=test_config["call_duration"]
    )
    
    assert result is True, f"基础呼叫失败: {test_config['caller_uri']} -> {test_config['callee_uri']}"
```

## 12. 部署和运行

### 12.1 系统要求
- Python 3.7或更高版本
- Windows/Linux/macOS操作系统
- 网络连接（用于测试SIP服务器）

### 12.2 安装步骤
1. 安装Python依赖：`pip install -r requirements.txt`
2. 配置系统参数：修改config/config.ini文件
3. 确保SIP服务器可用

### 12.3 运行方式
- 直接运行：`python -m pytest AutoTestForUG/pytest_sip_tests/ -v -m sip_basic`
- 使用执行器：`python AutoTestForUG/pytest_sip_tests/test_executor.py`
- 从YAML配置运行：`python AutoTestForUG/pytest_sip_tests/test_executor.py --config test_cases_yaml_example.yaml`
