# SIP自动化测试平台 - 项目计划（pytest架构版）
日期: 2026-01-08

## 项目目标调整
采用"开源测试框架 + 轻量领域层（DSL/adapter）"的混合方案，以pytest作为执行引擎，实现SIP自动化测试平台。重点优先实现最基本的SIP通话测试功能，确保能够完成简单的呼叫建立、通话和挂断流程。

## 新架构设计

### 核心架构
- **执行引擎**: pytest框架及其生态系统（pytest-xdist, pytest-rerunfailures, allure-pytest）
- **领域层**: SIP DSL（领域特定语言）用于描述SIP呼叫流程
- **测试定义**: YAML/JSON格式的测试用例定义
- **资源管理**: pytest fixtures管理端口池、客户端实例等共享资源

### 关键组件
- **Fixtures层**: 端口池管理、SIP客户端工厂、测试环境设置
- **DSL层**: SIP呼叫流程定义、消息验证器、测试场景构建器
- **解析器层**: YAML/JSON测试用例解析器
- **报告层**: 多格式测试报告生成器
- **执行器层**: 测试执行器，集成pytest运行

## 技术栈

### 测试框架
- **pytest**: 核心测试执行引擎
- **pytest-xdist**: 并行测试执行
- **pytest-rerunfailures**: 失败重试
- **allure-pytest**: 报告生成

### 领域特定语言
- **SIP DSL**: Python API用于描述SIP呼叫流程
- **YAML/JSON解析**: 测试用例定义格式

### 资源管理
- **Fixtures**: 端口池、客户端实例、测试环境管理

## 资源需求
- 1台SIP服务器用于测试
- 基础的网络环境
- Python 3.7+ 环境
- pytest相关依赖包
